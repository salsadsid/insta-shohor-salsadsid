Frequently Asked Questions (FAQ)

Q. How does JavaScript Works ?

A.JavaScript is a synchronous single-threaded programming language. That means it can execute one line at a time in a specific order. It is also a high level language. That means JavaScript stays away from computer. When a website loaded ,a thing called DOM(Document Obejct Model) will be created. Browser generate a DOM after grabing the HTML and CSS. Then the JavaScript code start working. JavaScript can change the DOM. JavaScript can run in every modern browsers. Every browser have their own JavaScript engine. Google Chrome uses the powerful JavaScript engine called V8.

Every engine mainly have two components.

  1. Memory heap
  2. Call stack
Memory heap stores data in a specified way. Every function in the call stack is called a stack frame. Code executes one by one in the call stack. After finishing all stack frames the call stack becomes empty.

Q. What does event loop do ?

A. There is a problem when a function in the call stack takes a huge amount of time. Like rendering weather conditions with a huge number of parameters. Code executes one by one. But when it takes a huge amount of time, it's getting blocked. There is a solution to this problem: asynchronous callbacks done by the event loop. If a function takes a huge amount of time, the function is sent to the task queue. Regular code from the call stack is executed in a regular way. The event loop observes the call stack. When the call stack is empty, the Event loop grap function from the queue and sent it to the queue. Event loop processing events in a loop.

Call stack empty => Pushing code from task queue to call stack => call stacks execute the code => again call stack empty

This is how event loop works.

Q. Difference between local storage & session storage ?

A. Local storage and session storage, both are web applications. Those two types of storage can stores data. Data about user's according to the websites. Local storage stores data for a long time. Session storage stores data for a short time. Session storage's data will be vanished, When user leaves the website. But Local storage's data remains premanently. Local storage's data remains after the browser is restarted or even after OS reboot. Both storage uses same syntax. The only difference between those storage's is expiration time. Session storage get cleared when browser tab or browser closed. On the other hand, local storage stores data with no expiration date.
